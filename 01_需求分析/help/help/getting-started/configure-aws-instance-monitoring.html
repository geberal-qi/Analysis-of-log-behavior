如何配置AWS实例日志监控

程序配置功能如下所示:

AWS实例日志监控的必要条件：

  1. 从EventLog Analyzer服务器主机所给URL: http://www.manageengine.com/products/eventlog/91517554/EventLogAnalyzerJSP.jar

    下载EventLog Analyzer jar文件包(EventLogAnalyzerJSP.jar)。

  2. 停止EventLog Analyzer服务并替换<EventLog Analyzer Home>/lib文件夹下的可用文件

  3. 重启EventLog Analyzer服务

  4. 确保你的Windows EC2实例可以从事件日志分析器访问服务器

程序安装

  • 从给定的URL: http://www.manageengine.com/products/eventlog/91517554/EventLogAgent.msi，下载EventLog Analyzer AWS代理(EventLogAgent.msi)到EC2 Windows实例。

  • 安装EventLog Analyzer服务器明细的代理

  • 出现欢迎屏幕和版权保护的信息

  • 确认代理安装要求

  • 输入服务器明细：服务器名称或服务器IP地址、,服务器数据库(PostgreSQL、MySQL、MS SQL)、服务器的协议(HTTP或HTTPS)、AWS实例(如果代理安装在AWS选择是，没有则选择否)、服务器端口(提到的HTTP/HTTPS服务器端口,默认端口是8400)

 

  • EventLog Analyzer代理作为AWS Windows实例服务器安装。

  • 检查服务是否正在运行

EC2服务器名称由提供的IP地址解析而来

你可以检查显示在“主机”子选项卡和代理管理设置页面的AWS实例。

五分钟后您可以查看AWS实例产生的报表。

注意:
  • 每个AWS Windows服务器实例上只能安装一个代理
  • 不能够把其他AWS服务器实例与AWS代理关联在一起
