EventLog Analyzer数据库的移动

对于EventLog Analyzer版本8.0（构建号8010）及之后的产品

  • 对于PostgreSQL、MySQL和MSSQL数据库，将EventLog Analyzer索引移动到另一个服务器
  • 移动PostgreSQL数据库到另一个服务器
  • 移动MSSQL数据库到另一个服务器

对于EventLog Analyzer版本8.0（构建号8000）及之前的产品

  • 移动MySQL数据库到另一个服务器

怎样找到构建号？

在EventLog Analyzer web客户端，点击页面右上角的帮助 > 关于链接，您可以上述提到的产品的构建号，此构建号就是当前EventLog Analyzer的构建号。

对于PostgreSQL、MySQL和MSSQL数据库，将EventLog Analyzer索引移动到另一个服务器

  • 从旧机器的位置复制索引（<EventLog Analyzer Home>\server\default\indexes）文件夹到新的服务器。

在新的服务器上完成安装之后：

  • 打开归档设置页面
  • 启用开启归档复选按钮
  • 修改日志索引位置并保存
注意：
  • 请检查您所运行的版本是否是最新版本，您可以在用户页面顶部的“关于”链接下查看这个信息。如果您运行的不是最新版本，请将您现有的版本升级到最新版本，然后在执行以下移动到新服务器的步骤。
  • 如果您要从32位服务器移动到64位的服务器，请联系我们的支持团队。


移动PostgreSQL数据库到另一个服务器

请参阅以下步骤，在新的服务器上保留旧的配置和数据。

将Eventlog Analyzer移动到另外一个服务器：

  1. 停止Eventlog Analyzer服务器/服务

  2. 检查任务管理器中是否存在进程“java.exe”、“postgres.exe”和“SysEvtCol.exe”，如果进程正在运行，则结束进程后继续以下步骤

  3. 将以下文件夹（包括文件和子文件夹）完全复制到另一个驱动器或映射的网络驱动器作为备份，以便在新机器上出现问题时，我们可以恢复设置和数据。

    1. 文件夹“pgsql”，位置：<Eventlog Analyzer Home>\

    2. 文件夹“Archive”，位置：<EventLog Analyzer Home>\server\default

    3. 文件夹“Indexes”，位置：<Eventlog Analyzer Home>\server\default

在对服务操作进行变更之前，请确认Eventlog Analyzer的安装版本已升级到了最新的版本，Eventlog Analyzer版本必须保持一致，才能使操作顺利执行。
  1. 请从以下链接下载Eventlog Analyzer的最新版本并进行安装：

    http://www.zohocorp.com.cn/manageengine/products/eventlog/index.html

  2. 当您在新的机器上安装完应用之后，请确保应用没有启动，如果Eventlog Analyzer启动了，则请将其关闭

  3. 重命名文件夹<EventLog Analyzer Home>\pgsql\data为“data_ori”

  4. 从旧机器复制pgsql 文件夹（位于<Eventlog Analyzer Home>\下）到新机器的相同位置

  5. 从旧机器复制Archive 文件夹（位于 <EventLog Analyzer Home>\server\default下） 到新机器的相同位置

  6. 从旧机器复制Indexes 文件夹（位于 <EventLog Analyzer Home>\server\default下） 到新机器的相同位置

请确保在执行这个操作时，Eventlog Analyzer在执行操作的系统上都没有运行。
  1. 在新机器上启动Eventlog Analyzer， 并检查数据和配置是否完整

移动MSSQL数据库到另一个服务器

  1. 停止Eventlog Analyzer服务器/服务

  2. 请从以下链接下载Eventlog Analyzer的最新版本并进行安装：

    http://www.zohocorp.com.cn/manageengine/products/eventlog/index.html

  3. 当您在新的机器上安装完应用之后，请确保应用没有启动，如果Eventlog Analyzer启动了，则请将其关闭

  4. 请参照配置MSSQL数据库主题中描述的步骤来配置MSSQL服务器凭证

  5. 在新机器上启动Eventlog Analyzer， 并检查数据和配置是否完整

移动MySQL数据库到另一个服务器

此部分的信息，仅适用于EventLog Analyzer版本8.0（构建号8000）及之前的产品。

请参阅以下步骤，在新的服务器上保留旧的配置和数据。

将Eventlog Analyzer移动到另外一个服务器：

  1. 停止Eventlog Analyzer服务器/服务

  2. 检查任务管理器中是否存在进程“java.exe”、“mysqld-nt.exe”和“SysEvtCol.exe”，如果进程正在运行，则结束进程后继续以下步骤

  3. 将以下文件夹（包括文件和子文件夹）完全复制到另一个驱动器或映射的网络驱动器作为备份，以便在新机器上出现问题时，我们可以恢复设置和数据。

    1. 文件夹“MySQL”，位置：<Eventlog Analyzer Home>\

    2. 文件夹“Archive”，位置：<Eventlog Analyzer Home>\

    3. 文件夹“Indexes”，位置：<Eventlog Analyzer Home>\server\default

如果在旧服务器上的MySQL设置了密码

    1. startDB.bat和configureODBC.vbs，位置：<Eventlog Analyzer Home>\bin

    2. myodbc3.dll和myodbc3s.dll，位置：<Eventlog Analyzer Home>\lib

    3. mysql-ds.xml，位置：<Eventlog Analyzer Home>\server\default\deploy

在对服务操作进行变更之前，请确认Eventlog Analyzer的安装版本已升级到了最新的版本，Eventlog Analyzer版本必须保持一致，才能使操作顺利执行。
  1. 请从以下链接下载Eventlog Analyzer的最新版本并进行安装：

    http://www.zohocorp.com.cn/manageengine/products/eventlog/index.html

  2. 当您在新的机器上安装完应用之后，请确保应用没有启动，如果Eventlog Analyzer启动了，则请将其关闭

  3. 重命名文件夹<Eventlog Analyzer Home>\MySQL为“MySQLori”

  4. 从旧机器复制MySQL文件夹（位于<Eventlog Analyzer Home>\下）到新机器的相同位置

  5. 从旧机器复制归档文件夹（位于 <Eventlog Analyzer Home>\下） 到新机器的相同位置

请确保在执行这个操作时，Eventlog Analyzer在执行操作的系统上都没有运行。
  1. 在新机器上启动Eventlog Analyzer， 并检查数据和配置是否完整
