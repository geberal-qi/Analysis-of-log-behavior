如何启动和关闭EventLog Analyzer

当你成功安装了EventLog Analyzer后，可以按照下面的步骤来启动EventLog Analyzer服务器。

  • 如何启动EventLog Analyzer服务/服务器？
  • 如何关闭EventLog Analyzer服务/服务器？

如何启动？

Windows程序：

  • 选择启动> 所有程序 > ManageEngine EventLog Analyzer <版本号> > EventLog Analyzer来启动服务器。
  • 或者，打开<EventLog Analyzer主目录>\bin，运行run.bat文件。

Windows服务：

确保EventLog Analyzer已经安装为Windows服务了。如果通过“一键安装”，默认安装成为了Windows服务。如果在“高级安装”的时候没有安装为Windows服务，按照转换产品为Windows服务中的步骤来操作。

当软件安装为服务之后，按照下面的步骤来启动Window服务：

  • 打开Windows的控制面板 > 管理工具 > 服务。右击ManageEngine EventLog Analyzer <版本号> ，选择启动。

Linux程序：

  • 打开<EventLog Analyzer主目录>/bin，运行run.sh文件。

当run.sh运行后，在命令行窗口中显示EventLog Analyzer个模块的启动信息。当所有模块启动后，将显示下面的信息：

Server started.
Please connect your client at http://localhost:8400

根据你指定的Web服务器端口，8400将替换为你指定的端口。

说明：如果EventLog Analyzer默认的syslog监听端口被占用，登录后将显示"不能绑定端口：<端口号>"信息。

Linux服务：

确保EventLog Analyzer已经安装为Linux服务了。如果通过“一键安装”，默认安装成为了Linux服务。如果在“高级安装”的时候没有安装为Linux服务，按照转换产品为Linux服务中的步骤来操作。

当软件安装为服务之后，按照下面的步骤来启动Linux服务：

运行命令：

/etc/init.d/eventloganalyzer start

查看EventLog Analyzer服务运行状态，运行命令：

/etc/init.d/eventloganalyzer status

显示状态为：

ManageEngine EventLog Analyzer 8.0 is running (<进程ID>)。

 

如何关闭？

按照下面的步骤来关闭EventLog Analyzer服务器。说明：当服务器成功关闭后，MySQL数据库连接将自动关闭，Eventlog Analyzer占用的所有端口将被释放。

Windows程序：

  • 打开开始 > 所有程序 > ManageEngine EventLog Analyzer <版本号>， 选择Shut Down EventLog Analyzer。
  • 或者打开<EventLog Analyzer主目录>\bin，运行shutdown.bat文件。确认后就可以关闭Evenglog Analyzer服务器了。

Windows服务：

如果安装并运行为Windows服务。
按照下面步骤停止Windows服务：

  • 打开Windows的控制面板 > 管理工具 > 服务。右击ManageEngine EventLog Analyzer <版本号> ，选择关闭。

Linux程序：

  • 打开<EventLog Analyzer主目录>/bin，执行shutdown.sh文件。

确认后就可以关闭Evenglog Analyzer服务器了。

Linux服务：

如果安装并运行为Linux服务。
运行以下命令停止Linux服务：

/etc/init.d/eventloganalyzer stop

输出信息：
Stopping ManageEngine EventLog Analyzer <版本号>...
Stopped ManageEngine EventLog Analyzer <版本号>

检查服务状态，运行命令：

/etc/init.d/eventloganalyzer status

输出信息：

ManageEngine EventLog Analyzer <version number> is not running.
